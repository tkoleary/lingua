When design meets internationalization

When you are designing and developing a website that will be used by people who speak multiple languages, or that has content that spans more than one language, a common problem is inline text from different scripts. Say for example your content is about Cambodia and the primary language of the page is English. You might want a heading that shows the place name like 'Ankor Wat' in both English and Khmer.

сЮвсЮДсЯТсЮВсЮЪсЮЬсЮПсЯТсЮП (Ankor Wat)

For the above to display correctly you need to either have a font loaded that will display the unicode characters, or (ugh) use an image.
